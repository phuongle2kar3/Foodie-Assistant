[BA Portfolio] Foodie Assistant – Nutrition Management & Smart Fridge Solution
📌 Project Overview

Foodie Assistant is an innovative mobile app designed for busy individuals and health-conscious users. The project focuses on optimizing meal preparation workflows by directly connecting with Smart Fridges for real-time ingredient management and personalized meal recommendations.

📂 Detailed Project Analysis
1. User Persona Analysis

I identified and analyzed 4 main user groups to ensure the product addresses their pain points:

Busy Professionals (Minh Hai – 35): Need speed and efficient use of available ingredients.
Fitness Enthusiasts (Phuong Anh – 20): Focused on Calories and Protein intake.
Professional Chefs (Nguyen Huong – 40): Require inventory and temperature management.
Students (Le Phuong – 22): Use the app independently (no IoT device needed) to browse recipes.
2. Business Processes

A key technical highlight is the IoT (Bluetooth) integration between the app and smart fridge. I established critical Business Rules (BR):

BR1: App automatically fetches fridge data via Bluetooth to minimize manual input.
BR2: Supports "Offline/No-device" mode for users without smart fridges.
BR4: Nutrition calculation algorithm based on actual ingredient weight.
3. Requirement Management (Product Backlog)

The requirement system was structured with:

User Stories: Written in the format "As a... I want... So that...".
Acceptance Criteria (AC): Clear standards (e.g., system response < 3s, kcal data accuracy).
Priority: Prioritized to optimize development resources.
4. Non-functional Requirements (Quality Attributes)

To ensure optimal user experience, I specified:

Performance: Response under 3 seconds; support 20,000 concurrent users.
Security: Full activity traceability and protection of personal data.
Compatibility: Smooth operation on both iOS and Android.
